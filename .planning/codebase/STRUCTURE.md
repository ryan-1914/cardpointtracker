# Codebase Structure

**Analysis Date:** 2026-02-28

## Directory Layout

```
cardtracker/
├── app.js                  # Main client logic (UI/events/state/IndexedDB)
├── index.html              # App shell and DOM structure
├── styles.css              # Design tokens and responsive styles
├── sw.js                   # Service worker and caching logic
├── manifest.webmanifest    # PWA metadata
├── assets/
│   └── icon.svg            # App icon
├── .planning/
│   └── codebase/           # Generated codebase mapping documents
└── .codex/                 # Local GSD workflows, tools, and templates
```

## Directory Purposes

**`assets/`:**
- Purpose: static media assets used by app shell and PWA metadata
- Contains: icon file(s)
- Key files: `assets/icon.svg`

**`.planning/`:**
- Purpose: planning artifacts generated by GSD workflows
- Contains: roadmap/requirements/phase docs and codebase map docs
- Key files: `.planning/codebase/*.md`

**`.codex/`:**
- Purpose: local automation resources for GSD commands
- Contains: skills, workflows, templates, helper scripts
- Key files: `.codex/get-shit-done/workflows/*.md`, `.codex/skills/*/SKILL.md`

## Key File Locations

**Entry Points:**
- `index.html`: browser document entry
- `app.js`: main runtime logic entry
- `sw.js`: service worker runtime entry

**Configuration:**
- `manifest.webmanifest`: PWA config
- `README.md`: local runtime instructions

**Core Logic:**
- `app.js`: card CRUD, reward ranking logic, render pipeline, service worker registration

**Testing:**
- No automated test files detected (`*.test.*` and `*.spec.*` absent)

## Naming Conventions

**Files:**
- Lowercase root asset names (`app.js`, `sw.js`, `styles.css`, `index.html`)
- Manifest uses descriptive lowercase filename (`manifest.webmanifest`)

**Directories:**
- Lowercase names (`assets`, `.planning`, `.codex`)

**Special Patterns:**
- Root-absolute asset paths in markup and service worker (`/app.js`, `/styles.css`)

## Where to Add New Code

**New Feature:**
- Primary logic: `app.js` (current pattern)
- Markup updates: `index.html`
- Styling: `styles.css`

**New Component/Module:**
- Current pattern: helpers/functions in `app.js`
- Recommended incremental modularization path: add `src/` and move logic by concern

**Utilities:**
- Shared helpers currently local to `app.js` (e.g., `escapeHtml`, `collectRewards`)
